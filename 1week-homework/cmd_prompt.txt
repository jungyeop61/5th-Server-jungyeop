도커 이미미지 빌드 명령어
$ docker image build -t helloworld:latest .

도커 컨테이너 실행 명령어
$ docker container run helloworld:latest

Node.js 버전 관리 도구인 nodebrew에서 안정 버전을 설치하도록 설정하는 명령
$ nodebrew install-binary stable

도커 이미지 받아오는 명령어
$ docker image pull gihyodocker/echo:latest

내려받은 이미지를 실행하는 명령어
$ docker container run -t -p 9000:8080 gihyodocker/echo:latest

CMD 명령 오버라이드
$ docker container run $(docker image build -q .) echo yay

도커 이미지 빌드하기
docker image build -t 이미지명[:태크명] Dockerfile의_경로
$ docker image build -t example/echo:latest .

생성된 이미지의 REPOSITORY, TAG, IMAGE ID, CREATED, SIZE 값들을 확인하는 명령어
$ docker image ls

이미지를 생성하고 컨테이너를 실행하는 명령어 (포어그라운드)
$ docker container run example/echo:latest

이미지를 생성하고 컨테이너를 실행하는 명령어 (백그라운드)
$ docker container run -d example/echo:latest

현재 실행중인 도커 컨테이너의 목록을 보는 명령어
$ docker container ls

실행한 컨테이너 정지하는 명령어
$ docker container stop $(docker container ls --filter "ancestor=example/echo" -q)

포트포워딩
$ docker container run -d -p 9000:8080 example/echo:latest

세세한 옵션을 볼 수 있는 도움말 명령어
$ docker help

이미지를 다루는 하위 명령
$ docker image --help

-f옵션 (Dockerfile이라는 이름으로 되어 있지 않은 파일명들은 -f옵션을 사용한다)
$ docker image build -f Dockerfile-test -t example/echo:latest .

-pull옵션 (매번 베이스 이미지를 강제로 새로 받아온다)
$ docker image build --pull=true -t example/echo:latest .

docker search - 이미지 검색
docker search [options] 검색_키워드
$ docker search --limit 5 mysql

docker image pull - 이미지 내려받기
docker image pull [options] 리포지토리명[:태그명]
$ docker image pull jenkins:latest

docker image ls - 보유한 도커 이미지 목록 보기
docker image ls [options] [리포지토리[:태크]]
$ docker image ls

docker image push - 이미지를 외부에 공개하기
docker image push [options] 리포지토리명[:태그]
$ docker image push stormcattest/echo:latest

docker container run - 컨테이너 생성 및 실행
docker container run [options] 이미지명[:태그] [명령] [명령인자...]
docker container run [options] 이미지ID [명령] [명령인자...]
$ docker container run -d -p 9000:8080 example/echo:latest

컨테이너 이름 붙이기
docker container run --name [컨테이너명] [이미지명]:[태그]
$ docker container run -t -d --name gihyo-echo example/echo:latest

docker container ls - 도커 컨테이너 목록 보기
docker container ls [options]
$ docker container ls

컨테이너 ID만 추출하기
$ docker container ls -q

컨테이너 목록 필터링하기
docker container ls --filter "필터명=값"
$ docker container ls --filter "name=echo1"

종료된 컨테이너 목록 보기
$ docker container ls -a

docker container stop - 컨테이너 정지하기
docker container stop 컨테이너ID_또는_컨테이너명
$ docker container stop 8842

docker container restart - 컨테이너 재시작하기
docker container restart 컨테이너ID_또는_컨테이너명
$ docker container restart echo

docker container rm - 컨테이너 파기하기
docker container re 컨테이너ID_또는_컨테이너명
$ docker container rm f66

현재 실행 중인 컨테이너 삭제 -f옵션
$ docker container rm -f f5f

docker container logs - 표준 출력 연결하기
docker container logs [options] 컨테이너ID_또는_컨테이너명
$ docker container logs -f $(docker container ls --filter "ancestor=jenkins" -q)

docker container exec - 실행 중인 컨테이너에서 명령 실행하기
docker container exec [options] 컨테이너ID_또는_컨테이너명 컨테이너에서_실행할_명령
$ docker container exec -it echo search

docker container cp - 파일 복사하기
docker container cp [options] 컨테이너ID_또는_컨테이너명:원본파일 대상파일
$ docker container cp echo:/echo/main.go .

prune - 컨테이너 파기하기
docker container prune [options]
$ docker container prune

prune - 이미지 파기하기
docker image prune [options]
$ docker image prune

prune - 시스템 파기하기
docker system prune [options]
$ docker system prune

docker container stats - 사용 현황 확인하기
docker container stats [options] [대상_컨테이너ID...]
$ docker container stats

docker-compose 명령으로 컨테이너 실행하기
$ docker-compose up -d

docker-compose 명령으로 컨테이너 정지
$ docker-compose down
